ilike_lowercase_pattern
ILIKE with a lowercase pattern — symmetric to the uppercase case. Catches the inverse bug if the serializer accidentally emitted a case-sensitive op.
ilike_no_match
ILIKE with a pattern that matches nothing — returns empty result.
ilike_optional_field
ILIKE on an Option<String> field — matches non-null values regardless of case; rows with NULL values are excluded since NULL ILIKE pattern is unknown.
ilike_uppercase_pattern
ILIKE with an uppercase pattern matches rows regardless of their case. On PostgreSQL this requires ILIKE; on SQLite/MySQL plain LIKE is already case-insensitive for ASCII, so the same query works.
