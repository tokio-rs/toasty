2//! Toasty is an async ORM for Rust supporting both SQL (SQLite, PostgreSQL,
3//! MySQL) and NoSQL (DynamoDB) databases.
4//!
5//! This crate is the user-facing API. It contains the database handle,
6//! query execution traits, and the types that generated code builds on. For
7//! a tutorial-style introduction, see the [Toasty guide].
8//!
9#![doc = include_str!(concat!(env!("OUT_DIR"), "/guide_link.md"))]
10//!
11//! # Modules
12//!
13//! ## [`db`] — database handle and connection pool
14//!
15//! The [`Db`] type is the entry point for all database interaction. It owns
16//! a connection pool and provides [`Db::builder`] for configuration. The
17//! module also contains [`Builder`](db::Builder) and the pool internals.
18//!
19//! ## [`schema`] — model, relation, and schema inspection
20//!
21//! The [`Model`](schema::Model) trait represents a root model that maps to a
22//! database table. It is implemented by `#[derive(Model)]` — users do not
23//! implement it manually. The module also contains [`Field`](schema::Field),
24//! which provides schema registration and runtime helpers for field types, and
25//! [`Auto`](schema::Auto), a wrapper for auto-generated values such as
26//! database-assigned IDs.
27//!
28//! The module also provides the types that represent associations between
29//! models: [`HasMany`](schema::HasMany), [`HasOne`](schema::HasOne), and
30//! [`BelongsTo`](schema::BelongsTo). These appear as fields on model structs
31//! and are populated through the generated relation accessors.
32//!
33//! The module also re-exports from `toasty-core` for inspecting the
34//! app-level and db-level schema representations at runtime.
35//!
36//! ## [`stmt`] — typed statement and expression types
37//!
38//! Contains the typed wrappers around the statement AST:
39//! [`Query`](stmt::Query), [`Insert`](stmt::Insert),
40//! [`Update`](stmt::Update), [`Delete`](stmt::Delete), and
41//! [`Statement`]. Also includes expression helpers like [`Expr`](stmt::Expr),
42//! [`Path`](stmt::Path), and the [`in_list`](stmt::in_list) function.
43//! Generated query builders (e.g. `find_by_*`, `filter_by_*`) produce these
44//! types.
45//!
46//! # Key traits
47//!
48//! - [`Model`](schema::Model) — a root model backed by a database table.
49//!   Implemented by `#[derive(Model)]`.
50//! - [`Embed`](schema::Embed) — an embedded type whose fields are flattened
51//!   into the parent model's table. Implemented by `#[derive(Embed)]`.
52//! - [`Executor`] — the dyn-compatible interface for running statements.
53//!   [`Db`] and [`Transaction`] both implement it. The generic
54//!   [`exec`](Executor::exec) method on `dyn Executor` accepts any typed
55//!   [`Statement<T>`].
56//! - [`Load`](schema::Load) — deserializes a model instance from the database
57//!   value representation.
58//!
59//! # Other key types
60//!
61//! - [`Transaction`] / [`TransactionBuilder`] — transactions with
62//!   auto-rollback on drop and nested savepoint support.
63//! - [`Page`](stmt::Page) — a page of results from a paginated query, with cursor-based
64//!   navigation.
65//! - [`Batch`](stmt::Batch) — groups multiple queries into a single round-trip.
66//! - [`Error`] / [`Result`] — re-exported from `toasty-core`.
67//!
68//! # Derive macros
69//!
70//! - [`#[derive(Model)]`](derive@Model) — generates the
71//!   [`Model`](schema::Model) impl, query builders, create/update builders,
72//!   relation accessors, and schema registration for a struct.
73//! - [`#[derive(Embed)]`](derive@Embed) — generates the
74//!   [`Embed`](schema::Embed) impl for a type whose fields are stored inline
75//!   in a parent model's table.
76//!
77//! # Feature flags
78//!
79//! Each database driver is behind an optional feature flag:
80//!
81//! | Feature        | Driver crate                 |
82//! |----------------|------------------------------|
83//! | `sqlite`       | `toasty-driver-sqlite`       |
84//! | `postgresql`   | `toasty-driver-postgresql`   |
85//! | `mysql`        | `toasty-driver-mysql`        |
86//! | `dynamodb`     | `toasty-driver-dynamodb`     |
87//!
88//! Additional feature flags: `rust_decimal`, `bigdecimal`, `jiff` (date/time
89//! via the `jiff` crate), and `serde` (JSON serialization support).
90//!
91//! # Other crates in the workspace
92//!
93//! `toasty` depends on several internal crates that most users do not need
94//! to use directly:
95//!
96//! - **`toasty-core`** — shared types used across the workspace: the schema
97//!   representations (app-level and db-level), the statement AST, the
98//!   [`Driver`](driver::Driver) trait, and [`Error`] / [`Result`].
99//! - **`toasty-macros`** — the proc-macro entry points and the code generation
100//!   logic they call.
101//! - **`toasty-sql`** — serializes the statement AST to SQL strings. Used by
102//!   the SQL driver crates.
103//! - **`toasty-driver-*`** — one crate per database backend.
