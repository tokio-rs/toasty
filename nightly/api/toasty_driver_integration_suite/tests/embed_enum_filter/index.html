create_and_get_data_enum
Creates records with different data-carrying enum variants and retrieves them by primary key, verifying enum values round-trip correctly. This exercises the same create + read path on all drivers including DynamoDB.
filter_data_enum
Filtering by a data-carrying enum value using a SQL WHERE clause. DynamoDB does not support arbitrary filter predicates, so this is SQL-only.
filter_data_enum_by_variant
Filtering by variant alone (discriminant-only check) using is_{variant}().
filter_enum_variant_with_partition_key
Filters by enum variant using is_{variant}() combined with a partition key. The partition key satisfies DynamoDB’s index requirement; the variant check becomes a FilterExpression applied server-side after the key lookup.
filter_unit_enum_by_variant
Filtering a unit-only enum by variant using is_{variant}().
