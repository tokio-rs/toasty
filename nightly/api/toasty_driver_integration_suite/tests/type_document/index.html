Module type_document 

Source
Expand description

Tests for #[document] storage of embedded types: a bare #[derive(Embed)] struct stored as one document column, and a Vec<T> of such structs stored as a JSON array of objects (jsonb on PostgreSQL, JSON1 text on SQLite). Each struct is encoded as a JSON object keyed by the embed’s field names.

This increment covers storage, encoding, whole-value CRUD, stmt::push onto a document collection, and nested-path filtering on bare struct embeds. #[document] on a Vec element predicate (.any() / .all()), partial! containment, and per-element mutation are not yet implemented. Backends without document support are gated out via requires(document_collections); the negative schema-build path has a dedicated requires(not(document_collections)) test.

struct_embed_bytes_document_rejected
A Vec<u8> leaf in a #[document] is rejected at schema-build: JSON has no binary representation, so the value could never round-trip through the document encoding.
struct_embed_temporal_create_get
Temporal leaves inside a #[document] round-trip through INSERT and a fresh fetch on every backend with document storage — including backends without native temporal column types. Regression test: the SQLite driver built toasty-sql without its jiff feature, so the JSON codec had no temporal arms and encoding a temporal document leaf panicked at insert.
struct_embed_whole_value_filter_rejected
Comparing a #[document] field to a whole embed value is not yet supported (document value equality is planned — see the design doc). It must fail with a clear unsupported_feature error rather than panicking inside the engine’s type inference.
