Module relation_via_mutation_runtime_error 

Source
Expand description

Mutation methods (insert, remove) on a multi-step relation traversal are not supported. The check is at exec time — the type system allows the call but the executor returns an error.

Before unifying Many, One, and OptionOne into a single per-model Query<T>, calling .insert() on a via relation was a compile error because Many<Via> had no such method. The unified design moves that rejection to runtime so the surface area stays small.
