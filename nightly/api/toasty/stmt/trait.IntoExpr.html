Trait IntoExpr 

Source 
pub trait IntoExpr<T> {
    // Required methods
    fn into_expr(self) -> Expr<T>;
    fn by_ref(&self) -> Expr<T>;
}
Expand description

Convert a value into an Expr<T>.

This trait is the primary way Toasty coerces Rust values into query expressions. It is implemented for all scalar types (i64, String, bool, uuid::Uuid, …), Option<T>, tuples, slices, Vec, and arrays.

Generated code uses IntoExpr bounds on filter and setter methods so that callers can pass either a raw value or an already-constructed Expr:

// Both &str and Expr<String> implement IntoExpr<String>:
let _e1: Expr<String> = "Alice".into_expr();
let _e2: Expr<String> = Expr::<String>::from_untyped(
    toasty_core::stmt::Value::from("Bob"),
).into_expr();

§Implementing for custom types

If you have a newtype that wraps a supported scalar, implement IntoExpr by converting to the inner type first, then calling Expr::from_value.

Required Methods§

Source

fn into_expr(self) -> Expr<T>

Consume self and produce an Expr<T>.

§Examples
use toasty::stmt::{Expr, IntoExpr};

let _expr: Expr<i64> = 42_i64.into_expr();
let _expr: Expr<String> = "hello".into_expr();
Source

fn by_ref(&self) -> Expr<T>

Produce an Expr<T> from a reference without consuming self.

For Copy types this clones the value. For non-Copy types like String this clones the underlying data.

§Examples
use toasty::stmt::{Expr, IntoExpr};

let value = 42_i64;
let _expr: Expr<i64> = value.by_ref();
// `value` is still usable
assert_eq!(value, 42);

impl<T: IntoExpr<T>> IntoExpr<T> for Deferred<T>

Forwards the inner value’s expression encoding so a Deferred<T> field can be spliced into any site that accepts a T.

Deferred<T> is a load-state wrapper, not a value type, so the produced expression is Expr<T> (the value), never Expr<Deferred<T>>. Panics with the standard “deferred field not loaded” error if the value is in the unloaded state — matching .get() / .into_inner().
