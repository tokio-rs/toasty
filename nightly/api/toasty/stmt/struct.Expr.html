Struct Expr 

Source 
pub struct Expr<T> { /* private fields */ }
Expand description

A typed expression in the Toasty query language.

Expr<T> wraps an untyped AST expression node and tags it with a Rust type T that represents the expression’s value type. Common instantiations:

  • Expr<bool> — a boolean filter expression (comparisons, and, or, not).
  • Expr<String>, Expr<i64>, etc. — scalar value expressions.
  • Expr<Option<T>> — a nullable expression with is_none and is_some helpers.
  • Expr<List<T>> — a list expression (see Expr::list).

Expressions are built from [Path] comparisons, literal values via IntoExpr, and combinators like and and or.

pub fn cast<U>(self) -> Expr<U>

Re-tag this expression with a different type U.

This performs no runtime conversion — the underlying AST node is unchanged. Use this when the type system needs a different type tag but the expression itself is compatible (e.g., widening Expr<T> to Expr<Option<T>>).

§Examples
let expr: Expr<i64> = 42_i64.into_expr();
let optional: Expr<Option<i64>> = expr.cast();

pub fn and_all<E>(exprs: impl IntoIterator<Item = E>) -> Self
where E: IntoExpr<bool>,

Combine an iterator of boolean expressions with logical AND.

Returns true (no filter) when the iterator is empty.

§Examples
use toasty::stmt::Expr;

let conditions = vec![
    User::fields().name().eq("Alice"),
    User::fields().age().gt(18),
];
let combined = Expr::and_all(conditions);
